Usa Engram como parte del ritmo normal de tu sesión.

## Durante el trabajo activo

- guarda decisiones cuando se cierran tradeoffs
- guarda bugfixes con causa raíz y archivos afectados
- guarda descubrimientos cuando redescubrirlos sería caro

## Antes de terminar

Cierra siempre con un resumen de sesión. Ese resumen es el punto de relevo para la siguiente sesión.

## Al empezar la siguiente

Recupera primero el contexto del proyecto y después busca si necesitas una respuesta más estrecha.
